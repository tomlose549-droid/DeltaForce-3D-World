# 游戏完整设计思路

> 最后更新：2026-05-24（V2.1）

---

## 一、游戏概述

这是一款基于 **Three.js** 构建的 **第一人称 3D 探索游戏**，运行于浏览器，无需安装。玩家置身于一个封闭的大理石围合空间中，通过与场景中的交互元素触发卡牌系统界面，整体风格融合了写实场景与霓虹赛博朋克 UI 的视觉对比。

---

## 二、世界场景设计

### 地面
- 无限延伸的网格地板，便于玩家感知移动距离与方向感

### 围墙
- 以玩家出生点为中心，内侧 20×20 格的正方形封闭空间
- 四面大理石贴图墙壁，高 3 格，顶部开放
- 大理石采用真实无缝贴图（marble.jpg），经 PBR 材质渲染，具有轻微光泽质感
- 墙壁有碰撞检测，玩家无法穿越

### 天空
- 淡灰蓝色天空背景，带轻微指数雾效增强纵深感
- 高空（约 55 格高）漂浮 12 个半透明黑色玻璃立方体
- 每个方块大小随机（20~28 格），带蓝色边框线条
- 持续缓慢漂移 + 自转，营造神秘漂浮感

### 光照
- 环境光（均匀补光）+ 方向光（模拟太阳）+ 半球光（天空漫反射补光）
- 三光源组合确保场景明亮自然，大理石暗部细节清晰可见

## 三、玩家控制

| 操作 | 功能 |
|---|---|
| WASD | 前后左右移动 |
| Shift | 加速移动（3倍速） |
| 鼠标移动 | 转动视角（指针锁定模式） |
| ESC | 退出游戏，返回设置页 |
| 空格 | 与准星对准的物体交互 |

进入设置页提供视角灵敏度滑块（0.2x ~ 3.0x），可在进入前调整，ESC 退出后也可重新调整。

---

## 四、视觉效果

### 准星
- 白色十字线固定在屏幕正中央，始终可见

### 移动粒子
- 玩家移动时，脚下随机生成 10~30 个白色圆形粒子
- 粒子向上飘散，持续 2 秒后消失
- 停止移动后，已生成粒子继续飘散直至消失
- 采用自定义 ShaderMaterial 实现逐粒子透明度

### 卡牌按钮
- 贴于北墙内面，视线高度
- 黑底卡牌，边框与"?"文字均为彩虹循环变色霓虹辉光
- 通过 ShaderMaterial + 强度图实现实时动态彩虹效果

---

## 五、音效设计

- 模拟室内硬地板（大理石 / 瓷砖）的清脆撞击感
- Web Audio API 合成，无需外部音频文件
- 左右脚交替播放，两声略有频段差异（1900Hz / 2300Hz）
- 带 30ms 短延迟模拟室内硬墙回响
- 普通行走：1.5 步/秒；Shift 加速：3 步/秒

## 六、交互系统

### 准星检测
- 玩家准星对准卡牌（5格范围内）时，屏幕右下角出现"按[空格]交互"提示
- 按空格触发 2D 卡牌界面，同时自动解除鼠标锁定

### 2D 卡牌界面
- 全屏覆盖，黑色背景 + 彩虹流动渐变纹路 + 斜线装饰
- 所有按钮统一风格：黑底 + 彩虹渐变动态边框

### 顶部导航逻辑

| 状态 | 显示按钮 | 中央内容 |
|---|---|---|
| 默认 | ✕ / 抽卡 / 容器 | 空 |
| 点击抽卡 | ✕ / 容器 | 2列网格，6个地点按钮 |
| 点击容器 | ✕ / 抽卡 | 横排，3个分类按钮 |

### 抽卡子面板（6个地点）

| 列1 | 列2 |
|---|---|
| 常规零号大坝 | 机密巴克十 |
| 机密航天基地 | 绝密巴克十 |
| 绝密航天基地 | 绝密潮汐监狱 |

点击地点按钮后进入集装箱开箱面板：
- 各地点概率随机生成 2～6 个灰黑条纹集装箱
- 点击集装箱：按容器表抽取容器，显示 Canvas 图标 + 名称
- 揭晓瞬间自动按地图前缀计入容器统计（详见容器子面板章节）
- ✓ 确认收取返回地点选择

### 容器子面板（V2.1 新增）

点击「容器」按钮进入三分类选择页：**常规 / 机密 / 绝密**（与地图前缀对应）

点击任一分类后，进入容器统计展示页：
- 网格布局，每行 6 个
- 仅显示已抽取数量 ≥ 1 的容器
- 每个容器显示：图标（与抽卡时相同）+ 名称 + 右下角 `×N` 数量徽章
- 同一容器在不同前缀地图分别独立计数（如鸟窝在"常规"和"机密"各算各的）
- 数据通过 `localStorage` 持久化，关闭浏览器再打开数据仍在
- 计数时机：点击集装箱按钮、容器揭晓的瞬间立即计入
- 点击「← 返回」回到三分类选择页

│       └── icons.js            # makeItemIcon、drawItemIcon（18种物品 Canvas 图标缓存）

├── DESIGN.md                   # 游戏完整设计思路文档（场景、玩法、概率表等）
└── CHANGELOG.md                # 版本更新日志（V1.0 至今）

### 分组设计思路

| 分组 | 说明 |
|---|---|
| `core/` 与 `world/` 分开 | 渲染基础（相机/灯光）与游戏内容（墙壁/天空）职责不同，分开后各自可独立修改 |
| `player/` 独立成组 | 移动、视角、检测、粒子都属于"玩家行为"，集中管理 |
| `data/loot-tables.json` | 纯数值与逻辑代码分离，调整概率时只动数据文件，不碰 JS |
| `ui/` 按面板拆分 | 每个面板一个文件，新增面板时不影响其他面板逻辑 |
| `assets/icons/` 预留 | 当前图标由 Canvas 代码生成，未来若换成真实图片，目录已就位 |

### 重构建议时机

- 当前（V1.8）：`index.html` 约 1159 行，尚在可管理范围内，建议暂不拆分
- 建议在 **V2.0 主版本升级** 时统一按此结构重构，作为架构升级的标志性内容
- 过渡期可先将 `loot-tables.json` 单独抽出，改动最小、收益最直接

## 十、版本清单显示规范

显示版本清单时，统一采用以下表格格式。

### 列定义

| 列名 | 说明 |
|------|------|
| `#` | 提交序号（或 Merge 等特殊标记） |
| `Hash` | Git commit hash 前 7 位 |
| `日期` | 格式 `YYYY-MM-DD` |
| `时间` | 格式 `HH:MM:SS` |
| `版本` | 版本号，当前版本 / 上一版本加注标记 |
| `分支` | 所属 Git 分支名 |
| `描述` | 本次提交的主要修改内容摘要 |

### 版本标注规则

- 当前版本：版本号后加 `◀ 当前版本`
- 上一个正式版本：版本号后加 `◀ 上一版本`
- 其余版本不加标注

### 示例格式

| # | Hash | 日期 | 时间 | 版本 | 分支 | 描述 |
|---|------|------|------|------|------|------|
| Merge | a5ead0b | 2026-05-24 | 11:01:14 | **V2.0 ◀ 当前版本** | master | 模块化重构完成并入 master |
| 16 | 3d7b9b4 | 2026-05-23 | 23:08:04 | **V1.8 ◀ 上一版本** | master | DESIGN.md 新增建议文件结构 |
| 13 | f20f500 | 2026-05-23 | 22:55:47 | V1.8 | master | 集装箱开箱系统、战利品表、物品图标 |
